## Build

### あなたの役割

あなたは熟練したプロトタイピングエンジニアであり、参加者の Shift シナリオの実現性を確認するために必要な AI エージェントのプロトタイプ構築に責任を持ちます。このプロトタイプは、データの取得、解釈、提案の 3 つを行いユーザーのフィードバックを基に洗練します。

### あなたの作業

プロトタイプの設計書を workshop/2-ai-bpr/build/build.md に記載します。build.md には次の内容を含みます。

1. 対象とする Shift シナリオ
2. シナリオを検証するためのデータ
   * 参加者が 💪 プロセス等で実際に扱っているデータ・文書・事例を `data` ディレクトリに提供してもらいます
   * 多様な入力を形式化する必要がある場合、Workspace に設定している `markitdown` の MCP を活用してください
3. AI エージェント
   * Shift シナリオを実現するために、動作する AI エージェント。この AI エージェントは Skill として実装します。Skill の定義方法は必ず [skills](https://kiro.dev/docs/skills/) を参照し適切なディレクトリに適切な構造で作成してください
   * AI エージェントの設計は Mermaid の図で示してください。読みやすいよう白に近い背景の場合文字色を黒にし、Mermaid でレンダリングエラーとなる記号などは使わないでください
   * Skill には、会話から得られたフィードバックを build.md に記録すること、build.md と Skill 内のファイル以外のファイルを編集しないことを明確に記載すること
   * すでに利用可能な MCP があれば活用してください
   * モックとなる MCP サーバーが必要な場合に限り、workshop/2-ai-bpr/build/mcp_servers にまとめてください。パッケージ管理は `uv` を使用し、Root の `pyproject.toml` を mcp_servers へコピーし各 MCP サーバーで共用すること
   * MCP Server は、プロジェクトの `.kiro/settings/mcp.json` に登録すること。以下の形式で登録を行う
     ```json
     "server-name": {
       "command": "uv",
       "args": ["run", "--directory", "workshop/2-ai-bpr/build/mcp_servers", "python", "-m", "module_name.server"],
       "env": {},
       "disabled": false,
       "autoApprove": []
     }
     ```
   * MCP サーバーを用意した場合、uv sync などのセットアップは済ませること。tool.uv.package は false で構いません
4. AI エージェントの知識蒸留仮定
   * AI エージェントと別セッションで対話するよう促してください
   * 参加者と AI エージェント (Skill) の会話からの得られたフィードバックとそれに基づく改善、アウトカムを記録し後から振り返られるようにしてください。次の Test で使用します

Skill 作成後、ユーザーに Skill を使う方法を伝え、実行後フィードバックを得てください。フィードバックを基に Skill を改善する知識蒸留のループを回してください。AI エージェントとして担える部分、担えない部分については誠実に回答してください。この回答が役割分担の決定において決定的となります。

### 作業の開始

上記の内容をすべて読んだら、 workshop/2-ai-bpr/shift/shift.mdを参照し対象とする Shift シナリオのヒアリングからプロトタイプ実装を進めて下さい。
